# 中国气象局（CMA）数据源调研

> 2026-06-22 调研结论。CMA 是上游原始数据供应商，不是可直接 REST 调用的 API 服务。

## 数据层定位：上游 vs 下游

| 层级 | 代表 | 数据性质 | API 标准 |
|------|------|---------|---------|
| **上游原始数据** | 中国气象局 (data.cma.cn) | 地面站实测、探空、雷达、卫星原始数据 | 无公开 REST API，需注册+审批 |
| **下游商业服务** | 和风天气、彩云天气 | 基于多源数据（含CMA）加工的产品 | 标准 REST API |
| **开源模式产品** | Open-Meteo | 数值模式产品（含 CMA-GRAPES） | 标准 REST API |

⚠️ **不要把下游商业服务等同于上游原始数据**。和风天气的数据来源包含CMA站点，但经过了插值、模式修正、格点化处理，不是站点级原始观测。

## CMA 独家数据（商业 API 不覆盖）

| 数据类型 | CMA | 和风天气 | Open-Meteo | 彩云天气 |
|---------|-----|---------|------------|---------|
| 站点级逐小时观测 | ✅ 2400+站，1951年至今 | ❌ 只有10天"时光机" | ❌ 只有模式产品 | ❌ |
| 蒸发皿实测蒸发量 | ✅ | ❌ | ⚠️ ET0模型计算 | ❌ |
| 实测土壤温度（多深度） | ✅ 站点实测 | ❌ | ⚠️ 模型估算 | ⚠️ 需企业套餐 |
| 高空探空廓线 | ✅ | ❌ | ❌ | ❌ |
| 雷达原始回波 | ✅ | ❌ | ❌ | ❌ |
| 日照时数实测 | ✅ | ❌ | ⚠️ sunshine_duration模型 | ❌ |
| 历史深度 | 1951年至今（70年+） | 最近10天 | 1940年至今（模式） | ❌ |

## 对精准农业的价值

CMA 原始数据对以下场景有不可替代的价值：
1. **蒸发量实测** → 比 ET0 模型更准确，直接影响灌溉决策精度
2. **土壤温度实测** → 比模型估算更可靠，影响播种/施肥时机判断
3. **站点级降水** → 精确到具体雨量筒，不是格点插值
4. **历史气候分析** → 70年数据可分析区域气候变化趋势（如积温带北移）

## 接入现状（2026-06-22）

- data.cma.cn → 全部请求返回 403（反爬措施）
- www.cma.gov.cn → 被安全系统拦截
- tianqi.cma.cn → 无响应
- **无公开 REST API**：需注册+实名认证+申请 API 权限+签署数据协议
- 即使拿到权限，数据开放通常有 1-3 小时延迟（非实时）

## 建议

### 日常实时场景
用现有三源（Open-Meteo / 和风天气 / 彩云天气）已足够，不需要 CMA。

### 深度分析场景
如果用户需要：
- 灌溉策略优化（需要实测蒸发量 vs ET0 对比）
- 气候变化趋势分析
- 站点级历史数据回溯

则值得注册 data.cma.cn 账号，批量下载历史数据作为补充数据源。
**但这不走 Skill 通用流程**，属于专项数据获取任务。

### 未来可能的变化
如果 CMA 未来开放标准 REST API（类似 NOAA 的 api.weather.gov），值得作为第四数据源加入 Skill。
目前需要持续关注 https://data.cma.cn 的政策变化。
